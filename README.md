## ✨ Ключевые возможности проекта

*   🚀 **Протокол VLESS TCP XTLS-Vision (Порт 443)** — золотой стандарт маскировки трафика. По умолчанию используется **надежный отпечаток TLS (`fp=ios`)**, обеспечивающий максимальную стабильность. Доступен интерактивный выбор отпечатков (включая `chrome`, `firefox`, `random` и др.). Везде принудительно используется **TLS 1.3** и чистый протокол **`alpn=http/1.1`** для 100% совместимости с любыми клиентами (Happ, Incy, Hiddify, V2rayN, Shadowrocket).
*   📱 **Динамический сервер подписок (Base64)** — разворачивает локальный легковесный Python-демон подписок. Импортируйте ссылку вида `https://ваш-домен.com/sub/UUID` в любой клиент, и настройки устройств обновятся автоматически! Специально оптимизировано для клиентов **Happ** и **Incy**: автоматическое включение Anti-DPI фрагментации (`tlshello`), умный автоконнект, интервалы обновлений и нативное оформление интерфейса профилей в приложении.
*   🌀 **Умный обход блокировок через Cloudflare WARP**:
    *   **Избирательный обход (Smart - по умолчанию)**: через WARP идут только заблокированные ресурсы (ChatGPT, Claude, Instagram, Netflix), сохраняя минимальный пинг и прямой выход вашего IP для остальных сайтов.
        *   ✅ *Плюсы:* Максимальная скорость, нет проблем с капчами и доступом к локальным сайтам, которые блокируют IP Cloudflare.
        *   ❌ *Минусы:* Ваш серверный IP виден сайтам. Если IP вашего хостинга (Hetzner, VDSina и др.) забанен самим сервисом, он не откроется.
    *   **Весь трафик (Full)**: весь исходящий трафик сервера оборачивается в сеть Cloudflare.
        *   ✅ *Плюсы:* Полная анонимность вашего VPS (все сайты видят "чистый" IP Cloudflare). Автоматически обходит строгие блокировки хостингов по IP-адресам.
        *   ❌ *Минусы:* Незначительно вырастает пинг. Google и другие сервисы могут чаще показывать проверку CAPTCHA ("Я не робот").
    *   **Гарантированный Warp-трафик**: Адреса популярных сервисов проверки IP (whoer, browserleaks, 2ip, ipleak и др.) и API-сервисов Google/Gemini автоматически идут через WARP в обоих режимах, позволяя мгновенно визуально проверить работоспособность.
    *   **Авто-обновление**: встроенная Cron-задача ежедневно обновляет списки блокировок с серверов GitHub.
    *   **Удобное управление**: возможность легкого переключения режимов и полного удаления WARP с сервера в один клик через меню `xry`.
*   🚦 **Интеграция RoscomVPN**: для клиентов Happ и Incy сервер автоматически отдает оптимизированный роутинг.
*   🇳🇱 **Автоопределение геолокации**: скрипт определяет страну вашего VPS и автоматически добавляет соответствующий флаг-эмодзи в имя подключения (например, `🇳🇱 VLESS-TCP`).
*   ⚡ **Ядерные оптимизации (BBR + TFO + KeepAlive)**: автоматическое включение Google BBR, агрессивного KeepAlive (300с) и TCP Fast Open на уровне ядра Linux снижает пинг, ускоряет загрузку данных и предотвращает зависание 'спящих' соединений мобильных провайдеров.
*   🛡️ **Имитация реального веб-ресурса (Decoy Page)**: при переходе на ваш домен посторонних лиц или DPI-сканеров, сервер демонстрирует правдоподобный рабочий интерфейс корпоративного портала Confluence (поддерживает автоопределение RU/EN языков браузера).
*   🔑 **Быстрая команда `xry`**: Забудьте длинные пути! Просто введите три буквы: **`xry`** в терминале вашего сервера для мгновенного вызова графической интерактивной панели управления.

## 🛠️ Быстрый старт (для новичков)

> [!IMPORTANT]
> Перед запуском убедитесь, что в панели вашего DNS-провайдера (например, Cloudflare) настроена **A-запись**, указывающая домен (например, `vpn.mycoolsite.com`) на публичный IP-адрес вашего сервера.

Выполните одну простую команду в терминале вашего сервера для удаленного запуска установки:

```bash
curl -fsSL "https://raw.githubusercontent.com/mvrvntn/vless-server-install/main/install_xray.sh?v=$(date +%s)" -o /tmp/install_xray.sh && sudo bash /tmp/install_xray.sh
```

## ⚡ Сравнение протоколов: Direct TCP (VLESS) vs CDN XHTTP (Тестовый режим)

> [!WARNING]
> **Протокол VLESS XHTTP в данном инсталляторе находится в тестовом (экспериментальном) режиме.**
> Настройки CDN-сетей (в частности, Cloudflare) могут сильно различаться, а поддержка XHTTP в мобильных клиентах пока еще внедряется. Используйте его как резервный вариант обхода блокировок.

В данном инсталляторе поддерживаются два режима работы. Вы можете использовать их параллельно (они не мешают друг другу):

| Характеристика | 🌐 Direct TCP (VLESS XTLS-Vision) | ⚡ CDN XHTTP (VLESS via CDN) [Тестовый режим] |
| :--- | :--- | :--- |
| **Принцип работы** | Прямое подключение клиента к IP-адресу вашего сервера. | Трафик идет через распределенную сеть CDN-провайдера (облако). |
| **Скорость и пинг** | **Максимальные**. Самый низкий пинг и наивысшая скорость скачивания. | **Средние/Хорошие**. Скорость и пинг зависят от пропускной способности CDN. |
| **Маскировка трафика** | Эмуляция обычного посещения сайта (TLS 1.3 + Vision padding). | Полная маскировка под обращение к легитимным веб-серверам CDN. |
| **Защита IP сервера** | **Нет**. Провайдеры могут заблокировать IP-адрес вашего сервера напрямую. | **Полная**. Клиенты видят только IP-адреса CDN (например, Cloudflare). IP сервера скрыт. |
| **Обход блокировок по IP** | Не спасает, если IP-адрес вашего VPS уже заблокирован в вашей стране. | **Идеально**. Обходит любые точечные блокировки IP-адресов серверов. |
| **Режим DNS в Cloudflare** | ⚠️ **DNS Only** (серое облако). | 🧡 **Proxied** (оранжевое облако). |
| **Сложность настройки** | **Очень простая**. Достаточно просто направить домен на IP сервера. | **Средняя**. Требует настройки проксирования, gRPC и TLS Full в панели CDN. |

### 👍 Плюсы и 👎 Минусы каждого протокола

#### 🌐 Direct TCP (VLESS XTLS-Vision) — Основной протокол по умолчанию
*   **👍 Плюсы:**
    *   **Максимальная скорость:** Скорость соединения ограничивается только пропускной способностью вашего VPS (легко выдает 500+ Мбит/с).
    *   **Минимальный пинг:** Прямой маршрут до сервера обеспечивает мгновенный отклик (важно для игр и звонков).
    *   **Простая настройка:** Никаких дополнительных доменов и настроек в сторонних сервисах.
*   **👎 Минусы:**
    *   IP-адрес сервера открыт для систем глубокого анализа трафика (DPI). В случае точечной блокировки IP вашего сервера протокол перестанет работать.

#### ⚡ CDN XHTTP (VLESS via CDN) — Протокол максимальной надежности (Тестовый режим)
*   **👍 Плюсы:**
    *   **Непробиваемый обход блокировок:** Трафик идет через гигантов вроде Cloudflare. Заблокировать их IP-адреса провайдеры не могут, так как это нарушит работу половины рунета.
    *   **Скрытие реального IP:** Никто не сможет узнать реальный IP-адрес вашего сервера по домену.
    *   **Спасение «забаненных» серверов:** Если IP вашего VPS уже заблокирован, XHTTP позволит продолжить им пользоваться.
*   **👎 Минусы:**
    *   **Ограничение скорости:** На бесплатном тарифе Cloudflare скорость обычно составляет 50–100 Мбит/с.
    *   **Повышенный пинг:** Трафик делает «крюк» через сервера CDN-провайдера.

## ⚙️ Пошаговая настройка CDN для VLESS-XHTTP (Тестовый режим)

> [!IMPORTANT]
> **VLESS-XHTTP находится в тестовом (экспериментальном) режиме!**
> Этот протокол использует новые механизмы транспорта в ядре Xray-core и полностью зависит от правильной настройки сторонней CDN (например, Cloudflare). Стабильность и скорость соединения могут уступать классическому Direct TCP. Используйте данный протокол в качестве резервного канала при блокировках.

Для работы протокола XHTTP через CDN вам необходимы **два** доменных имени (или поддомена):
1. **Основной домен (Direct TCP)** — например, `vpn.mydomain.com`. Должен быть направлен на сервер в режиме **DNS Only** (без проксирования). На него выпускается SSL-сертификат.
2. **CDN-домен (XHTTP)** — например, `cf-vpn.mydomain.com`. Направлен на тот же IP, но с включенным проксированием (**Proxied** / оранжевое облако).

### 1. Настройка в Cloudflare (Рекомендуется)
1. **DNS**: В панели Cloudflare добавьте новую A-запись для CDN-домена (например, `cf-vpn`), укажите IP вашего сервера и переключите облако в режим **Proxied** (оранжевое).
2. **SSL/TLS**: 
   * Перейдите во вкладку **SSL/TLS** -> **Overview**.
   * Выберите режим шифрования **Full** (Полный) или **Full (strict)** (Полный строгий). Режим *Flexible* (Гибкий) работать не будет!
3. **gRPC & WebSockets**:
   * Перейдите во вкладку **Network** (Сеть).
   * Убедитесь, что переключатели **WebSockets** и **gRPC** находятся в положении **On** (Включено). Это критически важно для прохождения XHTTP-пакетов.
4. **Смена домена на сервере**:
   * Запустите меню `xry`, выберите пункт `10` -> `2` и укажите ваш настроенный CDN-домен.
   * Скрипт автоматически обновит конфигурации. В меню просмотра кодов (пункт `1`) и в подписке появятся готовые ссылки для XHTTP.

### 2. Использование других CDN (CDNvideo, Beget, Timeweb, MWS и др.)
Вы можете использовать любую другую CDN-сеть, если она соответствует следующим техническим требованиям:
* **Поддержка gRPC и HTTP/2**: CDN должна уметь проксировать входящие HTTP/2 и gRPC-соединения.
* **Режим TLS**: CDN должна поддерживать проксирование SSL-трафика до вашего сервера с сохранением шифрования (режимы SSL-проброса или Full SSL, где CDN доверяет SSL-сертификату на вашем сервере).
* **Порты**: Для подключения клиентов используется порт `443` (HTTPS) на стороне CDN.

### 🔍 Решение проблем: Почему не работает VLESS-XHTTP?

Если после добавления CDN-домена подключение XHTTP не работает (бесконечное подключение, ошибки TLS или таймаут), проверьте следующие шаги по порядку:

#### 1. Проверьте версию и поддержку на стороне VPN-клиента
*   **Важно:** Старые версии клиентов не знают, что такое транспорт `xhttp` (он появился в Xray-core начиная с версии 1.8.0).
*   **Рекомендуемые клиенты:**
    *   **Happ** (версии 1.6.0+)
    *   **Incy** (версии 1.4.0+)
    *   **Hiddify** (последние версии)
    *   **v2rayN** (последние версии с ядром Xray v1.8.0+)
*   *В клиентах Shadowrocket, Streisand и некоторых других поддержка XHTTP может отсутствовать или требовать ручного включения в настройках.*

#### 2. Проверьте режим SSL/TLS в Cloudflare
*   Перейдите в панель Cloudflare -> вкладка **SSL/TLS** -> **Overview**.
*   Установите режим **Full** (Полный) или **Full (strict)** (Полный строгий).
*   ❌ Режим **Flexible** (Гибкий) работать **не будет**, так как Cloudflare будет пытаться слать на ваш сервер незашифрованные HTTP-запросы на порт 80, в то время как Xray ждет HTTPS-запросы на порту 443.

#### 3. Включите gRPC и WebSockets в Cloudflare
*   Перейдите во вкладку **Network** (Сеть) в Cloudflare.
*   Убедитесь, что переключатели **WebSockets** и **gRPC** находятся в положении **On** (Включено). gRPC жизненно необходим для работы транспорта XHTTP.

#### 4. Проверьте статус проксирования (Облако) в DNS
*   Основной домен (например, `vpn.mydomain.com`) -> **DNS Only** (Серое облако).
*   CDN-домен (например, `cf-vpn.mydomain.com`) -> **Proxied** (Оранжевое облако).
*   Если оба домена «серые» или оба «оранжевые», работать корректно не будет.

#### 5. Проверьте системные логи Xray на сервере
*   Зайдите на сервер и введите в консоли команду:
    ```bash
    xry
    ```
*   Выберите пункт **`5`** (`Просмотреть системные логи служб`) и выберите `Xray Core`.
*   Если при попытке подключения вы видите ошибки вида `invalid request` или `bad protocol`, убедитесь, что вы не используете шифрование/проксирование поверх неподдерживаемых портов или не включили `Flexible SSL` в Cloudflare.

> [!TIP]
> Если после обновления ссылки в клиенте не изменились, убедитесь что служба Python перезапущена, выполнив вручную: `systemctl restart xray-sub`.

## 🤝 Совместная работа рядом с AntiZapret-VPN (на одном сервере)

Вы можете абсолютно безопасно и эффективно использовать этот проект **на одном сервере вместе с AntiZapret-VPN**. Они будут отлично работать в параллели без малейших конфликтов:

### Почему они не мешают друг другу?
*   **AntiZapret-VPN** работает преимущественно по протоколу **UDP** (OpenVPN UDP на порту 1194 и WireGuard на портах 50000+).
*   Наш сервер **Xray** работает исключительно по протоколу **TCP** на портах **443** (для VLESS) и **80** (для выпуска SSL сертификатов).

### 📝 Пошаговая инструкция по параллельной установке:

#### Шаг 1. Освобождение порта 443 TCP в AntiZapret
По умолчанию AntiZapret занимает порт `443 TCP` в качестве резервного для OpenVPN. Чтобы отдать его более скоростному Xray VLESS:
1. Запустите стандартную установку **AntiZapret-VPN**.
2. В процессе установки или после неё остановите и навсегда отключите TCP-версию служб AntiZapret:
   ```bash
   sudo systemctl stop openvpn-server@antizapret-tcp openvpn-server@antizapret-no-vpn-tcp 2>/dev/null
   sudo systemctl disable openvpn-server@antizapret-tcp openvpn-server@antizapret-no-vpn-tcp 2>/dev/null
   ```
   *(При этом весь функционал обхода AntiZapret продолжит безупречно работать через основные протоколы UDP).*

#### Шаг 2. Установка VLESS TCP
Запустите команду установки нашего скрипта:
```bash
curl -fsSL "https://raw.githubusercontent.com/mvrvntn/vless-server-install/main/install_xray.sh?v=$(date +%s)" -o /tmp/install_xray.sh && sudo bash /tmp/install_xray.sh
```
Встроенный интеллектуальный инсталлятор сам проверит порты, обнаружит любые оставшиеся конфликты служб и поможет безопасно завершить настройку.

#### Шаг 3. Исключение конфликта фаерволов
Сервер AntiZapret использует сложные низкоуровневые правила `iptables NAT` для маршрутизации трафика и перехвата DNS. Системный фаервол **UFW** при активации жестко перекрывает и ломает эти цепочки (пользователи подключаются к OpenVPN, но у них нет интернета).

Если после установки Xray у вас перестал работать обход в AntiZapret:
1. Полностью отключите UFW:
   ```bash
   sudo ufw disable
   ```
2. **Обязательно перезагрузите сервер**, чтобы скрипты AntiZapret заново запустились и корректно прописали свои чистые маршруты в `iptables` без помех от UFW:
   ```bash
   reboot
   ```
*(Для Xray это абсолютно безопасно, так как он слушает порт 443 напрямую, и базовый фаервол Linux прекрасно с этим справляется).*

## 🛠️ Встроенная система диагностики (Troubleshooting)

Если у вас возникли проблемы с подключением, просто введите `xry` и выберите пункт **`7`** (`Запустить полную диагностику системы`).
Скрипт автоматически выполнит следующие проверки и покажет детальные результаты:
1.  **Сетевые порты 443 и 80**: проверит, какие именно процессы заняли данные порты и нет ли там конфликтов с OpenVPN/Nginx.
2.  **Демоны**: выведет статус активности служб `xray` и `xray-sub`.
3.  **DNS & Домен**: проверит локальный резолвинг домена и предупредит, если IP-адрес подменяется локальной заглушкой AntiZapret (`dnsmap` из подсети `10.224.x.x`).
4.  **Состояние WARP**: проверит целостность WireGuard-интерфейса и выполнит тестовый curl-запрос через WARP к Cloudflare для проверки доступности.
5.  **Сертификаты SSL**: проверит физическое наличие сертификатов Let's Encrypt и укажет точную дату окончания их действия.

## 📂 Структура проекта на сервере

После установки все ключевые компоненты распределяются по стандартным системным путям:

*   💾 **Конфигурационный файл Xray**: `/usr/local/etc/xray/config.json`
*   👥 **Конфиги пользователей**: `/etc/xray/client_configs/*.json`
*   🔑 **Быстрый ярлык**: `/usr/local/bin/xry` (ссылается на `/etc/xray/install_xray.sh`)
*   ⚙️ **Сервер подписок (Python)**: `/usr/local/bin/xray_sub_server.py`
*   🔄 **Демон systemd подписок**: `/etc/systemd/system/xray-sub.service`
*   📅 **Авто-обновление обходов (Cron)**: `/etc/cron.d/xray-geoblock-update`
*   📝 **Журналы событий (Логи)**: `/var/log/xray/`

## 🛑 Полное удаление проекта с сервера
Если вы решите удалить Xray, просто запустите `xry` и выберите пункт **`9`** (`Полностью удалить всю установку Xray с сервера`). Скрипт деликатно удалит все созданные файлы, службы, cron-задачи и вернет систему в исходное состояние, не затронув другие настройки сервера.
